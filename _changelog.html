Versioning#

The H3 core library adheres to Semantic Versioning. H3-Py has a major.minor.patch version scheme. The major and minor version numbers of H3-Py are the major and minor version of the bound core library, respectively. The patch version is incremented independently of the core library.

We use this changelog structure.

Because H3-Py is versioned in lockstep with the H3 core library, please avoid adding features or APIs which do not map onto the H3 core API.
